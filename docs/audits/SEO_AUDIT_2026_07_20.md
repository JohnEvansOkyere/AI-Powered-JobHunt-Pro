# VeloxaHire SEO Audit — 2026-07-20

## Current baseline

- `/jobs` and `/jobs/[id]` are public and accessible without candidate authentication.
- `/remote-jobs` is a public, server-rendered landing page for remote-job search intent.
- Job detail pages already have dynamic titles/descriptions and `JobPosting` JSON-LD.
- The homepage and job catalogue link to individual job detail pages.
- The repository had no `robots.txt` or XML sitemap route.
- The public jobs API exposed non-archived records, including pending or incomplete records.
- A live Google search for `site:veloxahire.org` returned no indexed VeloxaHire pages in this audit. This is an observation, not proof that every URL is unindexed.

## First implementation slice

Implemented in this pass:

- Added `frontend/app/robots.ts` with public crawl rules and a sitemap declaration. Authenticated/transactional paths are currently disallowed here; add explicit `noindex` route metadata before changing those rules.
- Added `frontend/app/sitemap.ts` for the homepage, public jobs catalogue, legal pages, and eligible job detail URLs.
- Added a lightweight backend sitemap projection that includes only processed jobs with a posting date, description, and apply/source URL.
- Added canonical URLs for the homepage, `/jobs`, and job detail pages.
- Changed incomplete/unprocessed job detail pages to `noindex` and emit no `JobPosting` JSON-LD.
- Improved the root title/description and corrected the Open Graph image alt text.
- Escaped JSON-LD delimiters before embedding job text in the HTML document.

## Step-by-step SEO roadmap

1. **Technical discovery:** deploy and verify `/robots.txt`, `/sitemap.xml`, canonical tags, and public job pages in Google Search Console.
2. **Job index quality:** confirm the cleanup/archive process removes expired roles, then add accurate `validThrough` data where the source provides it. Never invent expiry dates.
3. **Search intent pages:** create useful, human-written landing pages for real demand such as remote jobs, Ghana tech jobs, entry-level tech jobs, and CV/job-search guidance. Each page needs unique evidence and a clear candidate action. `/remote-jobs` is now the first page in this cluster.
4. **Internal linking:** link those pages to relevant live jobs and back to the main catalogue; avoid indexable filter/query URL explosions.
5. **Trust and conversion:** publish transparent source, freshness, eligibility, privacy, and application guidance. Track clicks from Search to job detail and apply.
6. **Measurement loop:** review Search Console queries, impressions, indexing exclusions, JobPosting enhancements, and page experience monthly; improve pages based on observed demand rather than keyword stuffing.

## Verification checklist

- [ ] Production `NEXT_PUBLIC_APP_URL` is the canonical public domain.
- [ ] `https://veloxahire.org/robots.txt` returns the expected rules.
- [ ] `https://veloxahire.org/sitemap.xml` returns core pages and current eligible jobs.
- [ ] `https://veloxahire.org/remote-jobs` renders current remote listings and links to detail pages.
- [ ] Search Console property and sitemap are configured.
- [ ] A sample job passes the Rich Results Test and URL Inspection.
- [ ] An archived/unfinished job is not in the sitemap and is `noindex`.
- [ ] Search Console shows no unexpected blocked, duplicate, or soft-404 job URLs.
